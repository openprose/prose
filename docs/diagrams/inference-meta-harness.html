Inference Meta-Harness

A model-backed OpenProse graph is not one long prompt. The runtime plans the graph, launches one Pi session per selected node, persists each outcome as a run, and feeds accepted upstream artifacts into downstream sessions.

Meta-Harness Responsibilities

Plan current, stale, blocked, skipped
Invoke one selected node at a time, dependency ordered
Materialize run records, artifacts, traces, pointers
Validate required outputs, declared effects, policy labels
Propagate accepted artifacts become downstream bindings
Explain trace shows why each node ran and what it produced

Boundary

OpenProse orchestrates runs, not thoughts. Pi owns one model-backed node session; the meta-harness owns dependency order, stale checks, durable state, policy gates, and provenance. Model providers such as OpenRouter live inside the Pi profile.
