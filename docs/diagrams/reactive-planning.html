Reactive Planning and Selective Recompute

OpenProse does not need to replay an entire workflow every time an upstream value changes. The planner compares requested outputs against prior materializations and only selects the slice that matters.

Selective Recompute Example

summarize depends on `draft` only current
market-sync depends on `company`, freshness, external reads ready
requested output: `summary` planner drops unrelated recompute current

Planner states

  • current: prior run is still valid
  • ready: stale and safe to re-run
  • blocked: missing inputs or gated effect

What the planner compares

IR hash did the program semantics change?
source hash did the source materialization change?
input hashes did the caller or upstream bindings change?
freshness / deps did external-read windows or pinned packages move?
