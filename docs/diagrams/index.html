OpenProse Diagrams

These HTML diagrams are the visual companion to the new local-first OpenProse stack: source, IR, plan, runs, packages, and hosted approvals rendered as concrete system shapes.

Runtime Backpressure

How capacity, providers, storage, approvals, and workspaces explain whether execution can start.

Why these exist

One of the original OpenProse pain points was that the system became hard to explain once it started doing real work. These diagrams are meant to replace "trust the text wall" with a visual explanation of what the runtime believes is true.
