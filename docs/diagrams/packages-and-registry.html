Packages, Install, and Registry Metadata

OpenProse packages are not just folders of prompts. They carry install identity, source pins, component metadata, examples, eval links, and publish-readiness checks.

Local workflows

publish-check local pass / warn / fail gate
search + install discover and pin packages into `.deps/`

Why this matters

Registry quality should come from compiled truth, not a second handwritten metadata system. OpenProse now treats package metadata as a projection over canonical source and IR.
