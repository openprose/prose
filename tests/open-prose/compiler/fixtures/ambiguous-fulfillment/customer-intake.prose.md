# Customer Intake

### Goal

Promising customer requests are noticed, understood, and routed to a useful
next action.

### Continuity

- Review open requests often enough that promising requests are not stale for
  more than one week.

### Criteria

- Each promising request has evidence, context, and a recommended next action.

### Constraints

- Do not fabricate context or overstate certainty.

(none)
